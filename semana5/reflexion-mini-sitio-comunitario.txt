Escribe una reflexión de 8 a 10 líneas respondiendo:

¿Por qué un mini-sitio que consume una API puede ayudar a una comunidad a difundir información de forma responsable, organizada y verificable?

En tu respuesta menciona al menos cinco de estas ideas:

- proyecto único en GitHub
- rama main
- rama personal
- git pull origin main
- fast-forward
- async
- await
- fetch
- API
- JSON
- difusión responsable
- planificación del calendario editorial
- calendario editorial
- revisión de fuentes
- consistencia comunicativa
- moderación editorial básica
- lenguaje responsable y no estigmatizante
- participación ciudadana
- cultura de paz

Respuesta:

Un mini-sitio que se conecta a una central de información (API) ayuda a que la comunidad reciba siempre datos reales y verificados, 
evitando que circulen rumores o noticias falsas. Al usar un proyecto único en GitHub, todo el equipo trabaja de forma organizada: 
los profes mantienen la base oficial en la rama main mientras cada uno hace sus pruebas en una rama personal, lo que garantiza una 
consistencia comunicativa para que el sitio no se rompa ni pierda su mensaje.

Gracias a herramientas como fetch, async y await, el sitio carga la información de forma rápida y fluida, permitiendo que el 
calendario editorial se mantenga al día automáticamente. Este esfuerzo técnico facilita una moderación editorial básica, 
asegurando que los mensajes usen un lenguaje responsable y no estigmatizante. Al final, esta organización no solo entrega datos, 
sino que construye una cultura de paz donde la información es clara, segura y útil para todos.
